# 记录 Skills `ai-package.json` 决策

**Summary**
- 新增文档：`/Users/bytedance/dev/ai-skills/docs/ai-package-skills.md`
- 若根目录 `docs/` 不存在，先创建。
- 文档只记录 skills v1 决策，plugins 明确标记为后续再聊。

**Document Content**
- 说明 `ai-package.json` 是用户可编辑的 manifest，优先于 `skills-lock.json`。
- 顶层使用 `skills`，不用 tiktok 当前的单数 `skill`。
- 每个 skill entry 使用：
```json
{
  "source": "github:owner/repo",
  "version": "main@commitSha",
  "path": "skills/example-skill"
}
```
- `source` 是 locator string，v1 支持 `github:`、`gitlab:`、`marketplace:`、`file:`。
- `version` 对远程源必填，格式为 `<ref>@<commitSha>`；安装以 commit sha 为准。
- `path` 必填，显式指向包含 `SKILL.md` 的目录，不做自动发现。
- skill name 来自 `skills` map 的 key，可以和源码目录名不同。

**Important Notes**
- `skills-lock.json` 后续只作为兼容或安装状态记录，不作为主配置。
- tiktok 旧版 `ai-package.md` 偏 stateful，不采用。
- tiktok `ai-package-v2.md` 的 ownership/ejected 设计暂不进入 v1。
- plugins 暂不纳入本轮 schema，避免提前污染 skills 的 Interface。

**Validation Examples**
- 合法：`github:vercel-labs/skills` + `main@sha` + `skills/find-skills`
- 合法：`file:.` + `.agents/skills/grille-me`
- 非法：远程源缺少 `version`
- 非法：缺少 `path`
- 非法：`path/SKILL.md` 不存在
