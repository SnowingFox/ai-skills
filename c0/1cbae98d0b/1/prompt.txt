<p>This skill helps you discover and install skills from the open agent skills ecosystem.</p>
<h2>When to Use This Skill</h2>
<p>Use this skill when the user:</p>
<ul>
<li>Asks "how do I do X" where X might be a common task with an existing skill</li>
<li>Says "find a skill for X" or "is there a skill for X"</li>
<li>Asks "can you do X" where X is a specialized capability</li>
<li>Expresses interest in extending agent capabilities</li>
<li>Wants to search for tools, templates, or workflows</li>
<li>Mentions they wish they had help with a specific domain (design, testing, deployment, etc.)</li>
</ul>
<h2>What is the Skills CLI?</h2>
<p>The Skills CLI (<code>npx skills</code>) is the package manager for the open agent skills ecosystem. Skills are modular packages that extend agent capabilities with specialized knowledge, workflows, and tools.</p>
<p><strong>Key commands:</strong></p>
<ul>
<li><code>npx skills find [query]</code> - Search for skills interactively or by keyword</li>
<li><code>npx skills add &lt;package&gt;</code> - Install a skill from GitHub or other sources</li>
<li><code>npx skills check</code> - Check for skill updates</li>
<li><code>npx skills update</code> - Update all installed skills</li>
</ul>
<p><strong>Browse skills at:</strong> <a href="https://skills.sh/" target="_blank" rel="ugc nofollow noopener">https://skills.sh/</a></p>
<h2>How to Help Users Find Skills</h2>
<h3>Step 1: Understand What They Need</h3>
<p>When a user asks for help with something, identify:</p>
<ol>
<li>The domain (e.g., React, testing, design, deployment)</li>
<li>The specific task (e.g., writing tests, creating animations, reviewing PRs)</li>
<li>Whether this is a common enough task that a skill likely exists</li>
</ol>
<h3>Step 2: Check the Leaderboard First</h3>
<p>Before running a CLI search, check the <a href="https://skills.sh/" target="_blank" rel="ugc nofollow noopener">skills.sh leaderboard</a> to see if a well-known skill already exists for the domain. The leaderboard ranks skills by total installs, surfacing the most popular and battle-tested options.</p>
<p>For example, top skills for web development include:</p>
<ul>
<li><code>vercel-labs/agent-skills</code> — React, Next.js, web design (100K+ installs each)</li>
<li><code>anthropics/skills</code> — Frontend design, document processing (100K+ installs)</li>
</ul>
<h3>Step 3: Search for Skills</h3>
<p>If the leaderboard doesn't cover the user's need, run the find command:</p>
<pre class="language-bash"><code class="language-bash code-highlight"><span class="code-line">npx skills <span class="token function">find</span> <span class="token punctuation">[</span>query<span class="token punctuation">]</span>
</span></code></pre>
<p>For example:</p>
<ul>
<li>User asks "how do I make my React app faster?" → <code>npx skills find react performance</code></li>
<li>User asks "can you help me with PR reviews?" → <code>npx skills find pr review</code></li>
<li>User asks "I need to create a changelog" → <code>npx skills find changelog</code></li>
</ul>
<h3>Step 4: Verify Quality Before Recommending</h3>
<p><strong>Do not recommend a skill based solely on search results.</strong> Always verify:</p>
<ol>
<li><strong>Install count</strong> — Prefer skills with 1K+ installs. Be cautious with anything under 100.</li>
<li><strong>Source reputation</strong> — Official sources (<code>vercel-labs</code>, <code>anthropics</code>, <code>microsoft</code>) are more trustworthy than unknown authors.</li>
<li><strong>GitHub stars</strong> — Check the source repository. A skill from a repo with &lt;100 stars should be treated with skepticism.</li>
</ol>
<h3>Step 5: Present Options to the User</h3>
<p>When you find relevant skills, present them to the user with:</p>
<ol>
<li>The skill name and what it does</li>
<li>The install count and source</li>
<li>The install command they can run</li>
<li>A link to learn more at skills.sh</li>
</ol>
<p>Example response:</p>
<pre class="language-text"><code class="language-text code-highlight"><span class="code-line">I found a skill that might help! The "react-best-practices" skill provides
</span><span class="code-line">React and Next.js performance optimization guidelines from Vercel Engineering.
</span><span class="code-line">(185K installs)
</span><span class="code-line">
</span><span class="code-line">To install it:
</span><span class="code-line">npx skills add vercel-labs/agent-skills@react-best-practices
</span><span class="code-line">
</span><span class="code-line">Learn more: https://skills.sh/vercel-labs/agent-skills/react-best-practices
</span></code></pre>
<h3>Step 6: Offer to Install</h3>
<p>If the user wants to proceed, you can install the skill for them:</p>
<pre class="language-bash"><code class="language-bash code-highlight"><span class="code-line">npx skills <span class="token function">add</span> <span class="token operator">&lt;</span>owner/repo@skill<span class="token operator">&gt;</span> <span class="token parameter variable">-g</span> <span class="token parameter variable">-y</span>
</span></code></pre>
<p>The <code>-g</code> flag installs globally (user-level) and <code>-y</code> skips confirmation prompts.</p>

<h2>Tips for Effective Searches</h2>
<ol>
<li><strong>Use specific keywords</strong>: "react testing" is better than just "testing"</li>
<li><strong>Try alternative terms</strong>: If "deploy" doesn't work, try "deployment" or "ci-cd"</li>
<li><strong>Check popular sources</strong>: Many skills come from <code>vercel-labs/agent-skills</code> or <code>ComposioHQ/awesome-claude-skills</code></li>
</ol>
<h2>When No Skills Are Found</h2>
<p>If no relevant skills exist:</p>
<ol>
<li>Acknowledge that no existing skill was found</li>
<li>Offer to help with the task directly using your general capabilities</li>
<li>Suggest the user could create their own skill with <code>npx skills init</code></li>
</ol>
<p>Example:</p>
<pre class="language-text"><code class="language-text code-highlight"><span class="code-line">I searched for skills related to "xyz" but didn't find any matches.
</span><span class="code-line">I can still help you with this task directly! Would you like me to proceed?
</span><span class="code-line">
</span><span class="code-line">If this is something you do often, you could create your own skill:
</span><span class="code-line">npx skills init my-xyz-skill
